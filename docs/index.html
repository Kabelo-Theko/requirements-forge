BA Reference

A quick-reference card for Business Analysts: INVEST criteria, DoR checklist, and Truth Guard methodology.

INVEST User Story Criteria

Well-formed user stories satisfy all six INVEST dimensions. The DoR scorer checks each one automatically.

I
Independent
Can be developed and delivered without depending on another story.
N
Negotiable
The scope is a starting point, open to discussion between team and stakeholder.
V
Valuable
Delivers clear, specific value to a user or the business.
E
Estimable
The team can estimate the effort. Vague actions make this impossible.
S
Small
Fits in a sprint. The action should be a single, clear verb phrase.
T
Testable
Has acceptance criteria in Given / When / Then form.

Definition of Ready scoring

The DoR score (0-100) is computed entirely in your browser from these checks:

  • Role present (15 pts): "As a [specific role]"
  • Action present (15 pts): "I want [action]"
  • Value present (15 pts): "so that [value]"
  • Value is specific (15 pts): value is more than a generic placeholder
  • Action is a single verb phrase (15 pts): no conjunctions splitting two actions
  • No vague words (10 pts): none of: etc, some, fast, robust, user-friendly, asap, seamless, intuitive, nice
  • At least one acceptance criterion (15 pts): the story is testable

Truth Guard methodology

Each acceptance criterion is checked against the original transcript. The guard normalises both strings (lowercase, collapse whitespace), then searches for any run of 4 or more consecutive words from the criterion in the transcript. If found: verified (green). If not: unverified (red warning). This makes hallucinated requirements immediately visible.

Artefacts produced

  • Stakeholder map: who is in the room, what they want, what they own or are blocked by
  • User stories: INVEST-format "As a / I want / so that" with a short id
  • Acceptance criteria: Given / When / Then per story, Truth-Guard-verified
  • Out of scope list: items explicitly ruled out or not discussed
  • Open questions / parking lot: things that need an answer before work can start
  • Conflict log: where two stakeholders contradict each other
